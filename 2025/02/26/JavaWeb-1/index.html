前者控制网页的结构，后者控制网页的样式

内联标签

<span> 是HTML中的一个内联元素（inline element），通常用于对文档中的部分文本或内联内容进行分组或者应用样式。与块级元素不同，内联元素不会在前后自动换行，因此它非常适合用于需要在一行内展示的元素。

基本用法

在这个例子中，<span> 标签被用来给“可爱”和“活泼”两个词应用不同的颜色样式。由于 <span> 本身没有默认的视觉表现形式，我们需要通过CSS来定义其样式。

特点

  • 内联特性：<span> 不会打断文本流，适用于需要在行内进行样式修改或脚本操作的情况。
  • 无语义意义：与 <header>、<article> 等具有明确语义的标签不同，<span> 主要用于装饰性目的或JavaScript操作。
  • 灵活应用：可以结合CSS和JavaScript实现复杂的交互效果和视觉设计。

实际应用示例

  1. 样式控制：如上述例子所示，使用 <span> 可以针对文本的一部分应用特定的样式。
  2. 动态内容更新：结合JavaScript，可以使用 <span> 来标记页面上需要动态更新的部分。
  3. 辅助功能：虽然 <span> 没有固有的语义价值，但在某些情况下，可以通过添加 aria-* 属性来增强可访问性。

总之，<span> 是一个非常灵活且强大的工具，广泛应用于网页开发中，特别是在需要精细控制内容样式的场景下。

实现正文-布局

盒子模型

  • 盒子：页面中所有的元素（标签），都可以看做是一个 盒子，由盒子将页面中的元素包含在一个矩形区域内，通过盒子的视角更方便的进行页面布局

  • 盒子模型组成：内容区域（content）、内边距区域（padding）、边框区域（border）、外边距区域（margin）

盒子的大小，其实就包括三个部分： border、padding、content，而margin外边距是不包括在盒子之内的。

布局标签

  • 布局标签：实际开发网页中，会大量频繁的使用 div 和 span 这两个没有语义的布局标签。

  • 标签：~<div> <span>~

  • 特点：

    • div标签：

      • 一行只显示一个（独占一行）

      • 宽度默认是父元素的宽度，高度默认由内容撑开

      • 可以设置宽高（width、height）

    • span标签：

      • 一行可以显示多个

      • 宽度和高度默认由内容撑开

      • 不可以设置宽高（width、height）

布局实现

  • 第一步：需要将body中的新闻标题部分、正文部分使用一个 div 布局标签将其包裹起来，方便通过css设置内容占用的宽度，比如：65%。
  • 第二步：通过css为该div设置外边距，左右的外边距分别为：17.5%，上下外边距靠边展示即可，为：0%。

表单标签

那表单呢,在我们日常的上网的过程中,基本上每天都会遇到。比如，我们经常在访问网站时，出现的登录页面、注册页面、个人信息提交页面，其实都是一个一个的表单 。 当我们在这些表单中录入数据之后，一点击 “提交”，就会将表单中我们填写的数据采集到，并提交， 那其实这个数据呢，一般会提交到服务端，最终保存在数据库中

那其实，上述的整个窗口是一个表单，而表单是一项一项的，这个我们称为表单项 或 表单元素。

  • 表单场景: 表单就是在网页中负责数据采集功能的，如：注册、登录的表单。

  • 表单标签: <form>

  • 表单属性:

    • action: 规定表单提交时，向何处发送表单数据，表单提交的URL。
    • method: 规定用于发送表单数据的方式，常见为： GET、POST。
      • GET：表单数据是拼接在url后面的， 如： xxxxxxxxxxx?username=Tom&age=12，url中能携带的表单数据大小是有限制的。
      • POST： 表单数据是在请求体（消息体）中携带的，大小没有限制。

  • 表单项标签: 不同类型的input元素、下拉列表、文本域等。

    • input: 定义表单项，通过type属性控制输入形式
    • select: 定义下拉列表
    • textarea: 定义文本域

GET提交

表单编写完毕之后，通过浏览器打开此表单，然后再表单项中录入值之后，点击提交，我们会看到表单的数据在url后面提交到服务端，格式为：?username=Tom&age=12。

POST提交

表单编写完毕之后，通过浏览器打开此表单，然后再表单项中录入值之后，点击提交，我们会看到表单的数据在请求体当中提交到服务端

叨叨

为了更清晰地解释这个过程，我们可以详细描述一下使用GET方法提交表单时数据的流动过程，并说明如何更好地理解“获取”和“提交”的概念。

使用 GET 方法提交表单的数据流

  1. 用户填写表单并提交：

    • 用户在网页上填写表单字段（例如，用户名为 Tom，年龄为 12）。
    • 当点击提交按钮时，浏览器会将这些表单数据附加到URL的查询字符串中。

  2. 浏览器构建请求URL：

    • 如果表单的action属性为空或未指定，默认情况下，表单数据会被发送到当前页面的URL。
    • 浏览器将表单数据编码成查询字符串格式，并将其附加到目标URL后面。例如：

  3. 浏览器发送HTTP GET请求：

    • 浏览器向服务器发送一个HTTP GET请求，其中包含上述带有查询字符串的URL。

  4. 服务器处理请求：

    • 服务器接收到这个GET请求后，解析URL中的查询字符串部分（即 username=Tom&age=12）。
    • 服务器端脚本（如PHP、Python、Node.js等）可以读取这些参数，并根据它们执行相应的操作（例如，查询数据库、生成动态内容等）。

  5. 服务器响应并返回结果：

    • 服务器处理完请求后，生成一个HTTP响应，并将处理结果（通常是HTML页面或其他格式的数据）返回给客户端（浏览器）。
    • 浏览器接收并渲染这个响应，显示给用户。

更接近于GET的概念

确实，在这种情况下，虽然我们说“提交”数据，但实际上这个过程更像是“获取”数据的一种形式。具体来说：

  • 从用户的视角：用户通过表单输入数据，然后点击提交按钮，这看起来像是在“提交”数据。
  • 从技术的角度：由于使用的是GET方法，浏览器实际上是发送了一个HTTP GET请求来“获取”由服务器处理后的结果页面。在这个过程中，表单数据只是作为查询字符串的一部分传递给服务器，以帮助服务器生成正确的响应。

因此，可以说，使用GET方法提交表单的过程实际上是一个“获取”数据的过程，因为：

  • 数据通过URL传递，类似于用户手动在地址栏输入URL并访问特定资源。
  • 服务器根据这些数据生成新的内容，并返回给客户端，而不是直接存储或修改数据（尽管某些应用可能会基于这些参数更新数据库，但这通常不是GET请求的主要用途）。

总结

  • GET方法：主要用于获取数据，即使它被用于表单提交，本质上还是一个获取资源的过程。数据通过URL传递，适合小量且非敏感的信息。
  • POST方法：主要用于提交数据，尤其是涉及大量或敏感信息的情况，数据通过HTTP消息体传递，更加安全。
