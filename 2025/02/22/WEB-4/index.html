然后我要在这个部分补充一个东西，就是关于url路径。

如果 URL 以斜杠开头（例如 /jsp/getBook.jsp?id=...），这表示从网站的根目录开始查找资源。这意味着无论当前页面位于哪个目录下，请求都会从网站的根目录开始。

如果不加斜杠（例如 jsp/getBook.jsp?id=...），这表示相对路径，即相对于当前页面所在的目录进行查找。

通常，在大型项目中，为了保持一致性和避免潜在的路径问题，推荐使用绝对路径（即在 JSP 文件路径前加上斜杠）。这样可以使代码更加健壮，减少由于页面位置变动而引起的路径问题。

JavaScript逻辑

  • xmlHttp.onreadystatechange：设置一个回调函数，当XMLHttpRequest对象的状态发生变化时会调用这个函数。

  • xmlHttp.readyState

    ：表示请求的状态。值为4表示请求已完成。

    • 0: 请求未初始化
    • 1: 服务器连接已建立
    • 2: 请求已接收
    • 3: 请求处理中
    • 4: 请求已完成且响应已就绪

  • xmlHttp.status

    ：表示HTTP状态码。值为200表示请求成功。

    • 200: 请求成功
    • 404: 未找到资源
    • 500: 内部服务器错误

  • xmlHttp.responseText：包含服务器返回的响应文本内容。

  • document.getElementById("book").innerHTML = data;：将服务器返回的数据插入到页面中具有 id="book" 的元素内，从而更新该区域的内容。

request 是 Java Servlet API 中的一个核心接口，代表了客户端发送到服务器的 HTTP 请求。它是 javax.servlet 包的一部分，用于获取请求中的信息，如请求头、请求参数、属性等。

在 JSP 页面中，request 对象是隐式对象之一，这意味着你不需要显式地声明它就可以直接使用它。这个对象封装了当前的 HTTP 请求，允许开发者通过它来访问各种与请求相关的信息和数据。

request 对象的主要用途包括但不限于：

  1. 获取请求参数：可以通过 request.getParameter() 方法获取表单提交的数据或其他类型的请求参数（如 URL 参数）。例如，在你的代码片段中：

  2. 获取请求头信息：可以使用 request.getHeader(), request.getHeaders(), 或者 request.getHeaderNames() 来获取HTTP请求头信息。

  3. 获取请求方法：通过 request.getMethod() 可以得知发起请求所使用的HTTP方法（GET, POST等）。

  4. 获取请求URL和URI：使用 request.getRequestURL() 和 request.getRequestURI() 分别获取完整的URL和相对路径。

  5. 会话管理：通过 request.getSession() 获取或创建一个HttpSession对象，用于维护用户状态。

  6. 请求分发和转发：可以使用 request.getRequestDispatcher() 将请求转发给另一个资源（如Servlet或JSP页面）处理。

总结

  1. 用户操作：用户在前端页面点击某个书籍类别。
  2. AJAX请求：前端通过AJAX向服务器发送请求，传递所选类别的categoryID。
  3. 服务器处理：服务器接收请求，使用JDBC连接数据库，执行查询并生成HTML片段。
  4. 响应返回：服务器将生成的HTML片段作为响应返回给前端。
  5. 前端更新：前端接收到响应后，将新的HTML片段插入到指定位置，更新页面内容。

通过这种方式，实现了无需刷新整个页面即可动态加载和显示不同类别的书籍列表，提升了用户体验。
