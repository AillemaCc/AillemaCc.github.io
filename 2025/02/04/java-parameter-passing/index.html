形参and实参

这几乎是任何编程语言在涉及到函数部分时，最重要的一个概念。

因为函数方法的定义，很多时候涉及到参数。参数在java当中分为下面两种

  • 实参：用于传递给函数/方法的参数，必须有确定的值。
  • 形参：用于定义函数/方法，接收实参，不需要有确定的值。

值传递and引用传递

将实参传递的方法分为两种

  • 值传递：方法接受的是实参值的拷贝，会对要传递的实参创建一个副本
  • 引用传递：方法接收的直接是实参所引用的对象在堆中的地址，不会创建副本，对形参的修改将影响到实参。

总结：

值传递和引用传递的真正区别，不是传递的内容，而是实参到底有没有被复制一份给形参。我们举一个例子：

你有一把钥匙（实参），当你的朋友需要去你家的时候，你直接把你的这把钥匙给他，这就是引用传递。在这种情况下，如果你朋友把这个钥匙修改了，那么这把钥匙还给你的时候（函数结束）这把钥匙当然会保留你朋友的修改

但是如果你复刻了一把新钥匙，自己的钥匙还在自己手里。你朋友修改了新钥匙，那么也不会影响你手里的这把钥匙

为什么不引入引用传递呢

引用传递看似很好，能在方法内就直接把实参的值修改了，但是，为什么 Java 不引入引用传递呢？

注意：以下观点并非来自于 Java 官方：

  1. 出于安全考虑，方法内部对值进行的操作，对于调用者都是未知的（把方法定义为接口，调用方不关心具体实现）。你也想象一下，如果拿着银行卡去取钱，取的是 100，扣的是 200，是不是很可怕。
  2. Java 之父 James Gosling 在设计之初就看到了 C、C++ 的许多弊端，所以才想着去设计一门新的语言 Java。在他设计 Java 的时候就遵循了简单易用的原则，摒弃了许多开发者一不留意就会造成问题的“特性”，语言本身的东西少了，开发者要学习的东西也少了。
