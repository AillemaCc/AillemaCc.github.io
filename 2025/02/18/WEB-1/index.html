关于Servlet

大三下学年的第二个实验就涉及到创建一个servlet类，并将其部署到tomcat服务器当中，实现一定的功能

在这里我们先通俗的介绍一下这个技术

Servlet 是一种用于在服务器端处理客户端请求并生成响应的Java类。就相当于一个中间人，先接收来自浏览器的HTTP请求，然后根据请求的内容执行相应的操作，再返回一个响应给客户端。基本上执行的是web这一层的工作。

如果要详细解释一下他的工作流程，大概是这样的：

  1. 接收请求：当用户在浏览器中输入一个URL或提交一个表单时，浏览器会发送一个HTTP请求到服务器。这个请求会被传递给相应的Servlet进行处理。
  2. 处理请求：Servlet接收到请求后，会解析请求的内容，比如请求的类型（GET、POST等）、请求参数（如表单数据）等。然后根据这些信息决定如何处理这个请求。例如，查询数据库、调用其他服务、读取文件等。
  3. 生成响应：处理完请求后，Servlet会生成一个响应，通常是HTML页面、JSON数据或其他格式的数据。这个响应会被发送回客户端（浏览器），并在浏览器中显示出来。
  4. 结束交互

看到servlet的工作流程，不难联想到Spring Boot 中Controller 层。在 Spring Boot 应用中，Controller 层充当了 Servlet 的角色，接收 HTTP 请求并调用相应的 Service 方法。然后将 Service 返回的数据封装成 HTTP 响应返回给客户端。

古法引入

首先，需要下载 Servlet API 的 JAR 文件。

  • Maven Central Repository：访问 Maven Central 并搜索 javax.servlet-api 或 jakarta.servlet-api（取决于你使用的版本），然后下载对应的 JAR 文件。
  • Tomcat 安装目录：如果你已经安装了 Tomcat，可以在其 lib 目录下找到 servlet-api.jar 或 jakarta.servlet-api.jar 文件。

例如，对于 Tomcat 10 及以上版本（Jakarta EE 9+），你可以从以下链接下载：

  • Jakarta Servlet API (5.0.0)

对于 Tomcat 9 及以下版本（Java EE 8 及以下），你可以从以下链接下载：

  • Servlet API (4.0.1)

之后，需要将JAR文件添加到项目的类路径当中

在没有使用构建工具（如 Maven 或 Gradle）的情况下，需要手动将下载的 JAR 文件添加到项目的类路径中。具体操作如下：

使用 IDE（如 Eclipse 或 IntelliJ IDEA）

  1. 创建项目：在 IDE 中创建一个新的 Java Web 项目。
  2. 导入 JAR 文件：
    • Eclipse：
      1. 右键点击项目 -> Build Path -> Configure Build Path。
      2. 在 Libraries 标签页中，点击 Add External JARs...。
      3. 浏览并选择你下载的 servlet-api.jar 或 jakarta.servlet-api.jar 文件，然后点击 Apply and Close。
    • IntelliJ IDEA：
      1. 打开项目设置：File -> Project Structure。
      2. 在 Modules 中选择你的模块，然后点击 Dependencies 标签页。
      3. 点击 + 按钮，选择 JARs or directories...。
      4. 浏览并选择你下载的 servlet-api.jar 或 jakarta.servlet-api.jar 文件，然后点击 OK。
